# Offer Catcher - AI求职智能匹配平台方案说明

## 一、问题诊断

### 核心痛点
学生求职面临两大难题：
1. **岗位匹配效率低** - 在海量岗位中寻找匹配工作耗时巨大
2. **简历优化无方向** - 不清楚简历与岗位的匹配度，不知道如何优化

### 原始方案问题
- 单文件架构（720行app.py），可维护性差
- 无缓存机制，重复解析浪费时间
- 错误处理不友好，用户不知道如何解决
- 加载时间长（30-80秒），用户体验差

## 二、方案设计

### 产品定位
AI驱动的求职智能匹配平台，帮助学生高效匹配岗位并提升简历通过率。

### 核心功能
| 功能 | 输入 | 输出 | 价值 |
|------|------|------|------|
| 智能岗位匹配 | 简历+多个JD | 匹配排序+优化建议 | 找到最匹配岗位 |
| 智能面试准备 | 简历+1个JD | 面试题库+自我介绍 | 准备面试 |
| 候选人深度验证 | 简历+1个JD | 面试策略+问题设计 | 面试官验证 |

### 技术架构
```
前端(Flask模板) → 后端(Flask) → 解析层(MinerU) → AI层(MiMo)
                    ↓
               缓存层(文件缓存)
```

## 三、AI工具选型

### MinerU - 文档解析
- **选择理由**：开源OCR工具，支持PDF/图片解析，中文识别准确
- **使用场景**：简历和JD的文本提取
- **配置**：API调用，120秒超时，3次重试

### MiMo - AI推理
- **选择理由**：小米自研大模型，中文理解能力强，API稳定
- **使用场景**：结构化解析、匹配分析、面试题生成
- **配置**：mimo-v2.5-pro模型，4096 token限制

## 四、关键配置

### 缓存机制
- **缓存策略**：文件缓存，MD5作为key
- **缓存位置**：`cache/`目录
- **效果**：相同文件解析速度提升80%

### 错误处理
- **分层处理**：API层→解析层→UI层
- **友好提示**：具体错误信息+解决方案
- **重试机制**：一键重试按钮

### 进度显示
- **思维链模式**：按顺序显示每个步骤
- **实时反馈**：用户知道当前进度和预计时间

## 七、总结

Offer Catcher通过AI技术解决了学生求职的核心痛点：
1. **效率提升** - 缓存机制让重复解析瞬间完成
2. **体验优化** - 友好的错误提示和进度显示
3. **功能完整** - 覆盖求职全流程

项目采用模块化设计，易于维护和扩展，为后续功能迭代奠定了基础。
