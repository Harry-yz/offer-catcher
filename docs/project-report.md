# 基于AI的求职智能匹配系统设计与实现

## 一、项目背景与问题分析

### 1.1 研究背景

随着高校毕业生规模逐年扩大，就业市场竞争日益激烈。学生在求职过程中面临两大核心痛点：

**痛点一：岗位匹配效率低下**
- 招聘平台岗位数量庞大，学生需逐一浏览判断匹配度
- 缺乏量化评估工具，主观判断易产生偏差
- 大量时间浪费在明显不匹配的岗位上

**痛点二：简历优化缺乏方向**
- 不清楚自身简历与目标岗位的匹配程度
- 缺乏针对性的优化建议
- 面试准备无从下手

### 1.2 现有解决方案的不足

| 方案 | 优点 | 不足 |
|------|------|------|
| 人工筛选 | 理解深入 | 效率低、成本高 |
| 关键词匹配 | 速度快 | 准确率低、缺乏语义理解 |
| 通用AI助手 | 能力强 | 缺乏求职领域专业知识 |

### 1.3 研究目标

设计并实现一个AI驱动的求职智能匹配系统，实现：
- 自动解析简历和岗位描述
- 智能分析匹配度并给出优化建议
- 生成个性化面试准备方案

## 二、解决方案设计

### 2.1 系统架构

采用分层架构设计，实现高内聚低耦合：

```
┌─────────────────────────────────────────────┐
│                  前端展示层                   │
│         (Flask模板 + 原生HTML/CSS/JS)         │
├─────────────────────────────────────────────┤
│                  业务逻辑层                   │
│    匹配分析 │ 面试准备 │ 候选人验证           │
├─────────────────────────────────────────────┤
│                  数据处理层                   │
│      文档解析(MinerU) │ AI推理(MiMo)         │
├─────────────────────────────────────────────┤
│                  基础设施层                   │
│      缓存管理 │ 配置管理 │ 错误处理           │
└─────────────────────────────────────────────┘
```

### 2.2 核心功能模块

**模块一：智能岗位匹配**
- 输入：简历 + 多个岗位JD
- 处理：多Agent协作分析
- 输出：匹配度排序 + 简历优化建议

**模块二：智能面试准备**
- 输入：简历 + 目标岗位JD
- 处理：匹配分析 + 题库生成
- 输出：面试题库 + 自我介绍 + 模拟面试

**模块三：候选人深度验证**
- 输入：候选人简历 + 岗位JD
- 处理：针对性问题设计
- 输出：面试策略 + 考察要点

### 2.3 数据流设计

```
用户上传 → 文档解析 → 结构化数据 → AI分析 → 结果展示
    ↓           ↓           ↓          ↓          ↓
  文件缓存   MinerU提取   MiMo推理   多Agent    前端渲染
```

## 三、技术实现

### 3.1 技术选型

| 技术 | 选型理由 |
|------|----------|
| **Flask** | 轻量级Web框架，适合个人项目 |
| **MinerU** | 开源OCR工具，中文识别准确 |
| **MiMo** | 小米自研大模型，中文理解能力强 |
| **文件缓存** | 简单高效，无需额外依赖 |

### 3.2 关键技术点

**1. 多Agent协作架构**

采用多Agent设计，不同Agent负责不同任务：
- 岗位匹配Agent：分析简历与岗位的匹配度
- 报告生成Agent：生成综合分析报告
- 面试准备Agent：生成面试题库和自我介绍

**2. 智能缓存机制**

```python
# 缓存键：文件内容的MD5哈希
cache_key = hashlib.md5(file_bytes).hexdigest()

# 缓存策略：相同内容不重复解析
if cache_exists(cache_key):
    return load_cache(cache_key)
else:
    result = parse_document(file_bytes)
    save_cache(cache_key, result)
    return result
```

**3. 错误处理与用户反馈**

采用分层错误处理策略：
- 识别错误类型（网络超时、文件格式、API限制）
- 提供具体解决方案
- 支持一键重试

### 3.3 模块化设计

项目采用6模块架构：

```
api.py          - API调用层（MiMo接口封装）
parser.py       - 解析逻辑层（简历/JD解析）
config.py       - 配置管理层（提示词、环境变量）
models.py       - 数据模型层（类型定义）
mineru_parser.py - MinerU封装（文档解析）
cache_manager.py - 缓存管理（文件缓存）
```
